## Sizes

Please know that the `small` version is not really supposed to be used on touch devices to avoid misclicking. We recommend some blank space before and after the quantity selector to have the best user experience possible.

## Accessibility and semantic

<HintItem>
  The input field element must always have an attribute <strong>id</strong>
</HintItem>
<HintItem>
  Use the <strong>aria-label</strong> attribute on the input element to defines
  the accessible name of the quantity selector.
</HintItem>
<HintItem>
  Use the <strong>aria-valuenow</strong> attribute on the input element to
  indicate the current numeric value of the quantity selector.
</HintItem>
<HintItem>
  Use the <strong>aria-valuemin</strong> attribute on the input element to set
  the minimum value allowed for the quantity selector.
</HintItem>
<HintItem>
  Use the <strong>aria-valuemax</strong> attribute on the input element to set
  the maximum value allowed for the quantity selector.
</HintItem>
<HintItem>
  The buttons elements (mc-button) must always have an
  <strong>aria-controls</strong> attribute whose value corresponds to the value of
  the <strong>id</strong> attribute of the input field element.
</HintItem>
<HintItem>
  The buttons elements (mc-button) must always have an
  <strong>aria-label</strong> attribute whose value defines the accessible name for
  each increase and decrease button.
</HintItem>
<HintItem>
  The buttons elements (mc-button) must always have an{' '}
  <strong>aria-hidden="true"</strong> attribute to hide them for assistive
  technology users.
</HintItem>
