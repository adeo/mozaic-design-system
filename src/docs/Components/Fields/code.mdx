## Import

Import the settings, the text-input and the fields `scss` files.

```scss
@import 'settings-tools/_all-settings';

@include import-font-families();

@import 'components/_c.text-input';

@import 'components/_c.fields';

```

## Basic usage

To create a fields you have two main class to apply :
- `mc-fields` on container
- `mc-fields__label` on label

and reuse the `mc-text-input` class on the input

```html
<div class="mc-fields">
    <label class="mc-fields__label" for="inp">Label</label>
    <input id="inp" type="text" class="mc-text-input" placeholder="Empty" name="example" />
</div>
```

## Accessibility and semantic

### Always use a label with an input.

```html
<div class="mc-fields">
    <label class="mc-fields__label" for="myExample">My example</label>
    <input
        id="myExample"
        type="text"
        class="mc-text-input"
        placeholder="Empty"
        name="example"
    />
</div>
```

### Required field

If an input field is mandatory, then the `required` and `aria-required` attributes must be filled in.

```html
<div class="mc-fields">
    <label class="mc-fields__label" for="requiredField">Required field</label>
    <input
        id="requiredField"
        type="text"
        class="mc-text-input"
        placeholder="Is required"
        name="example"
        required
        aria-required="true"
    />
</div>
```

### Invalid field

A text field is invalid when its value does not match the expected type. When a field is invalid, the `aria-invalid` attribute must be added dynamically to the input field in order to indicate it to screen readers.

```html
<div class="mc-fields">
    <label class="mc-fields__label" for="invalidField">Invalid field</label>
    <input
        id="invalidField"
        type="text"
        class="mc-text-input is-invalid"
        placeholder="Is invalid"
        name="example"
        aria-invalid="true"
    />
    <span class="mc-fields__error-message">Error</span>
</div>
```
